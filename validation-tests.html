Part 4: Test Plan - Validation Testing

As part of the validation process, we will conduct tests to ensure proper functionality and expected behavior of the web application. The tests will be separated into 6 categories, each testing a separate functionality.

Usability Testing

Usability testing is a technique used in user-centered interaction design to evaluate a product by testing it on users. This can be seen as an irreplaceable usability practice, since it gives direct input on how real users use the system. This will ensure that the mobile app and website are easy to use and provide a satisfactory user experience to the customers. It will also provide information towards if users are able to complete discrete tasks successfully and identify how long it takes to complete each task.

Compatibility Testing

Compatibility testing is a part of non-functional testing conducted on application software to ensure the application's compatibility with different computing environments.This will check whether the system is capable of running on different hardware, operating systems, applications, network environments or mobile devices according to the requirements.

Interface Testing

An interface is a connection that integrates two components within a system. An interface can range from a Graphical User Interface (GUI) to APIs and server based communication between the database server to application server or client to web server. Interface Testing is defined as a software testing type which verifies whether the communication between two different software systems is done correctly.

Services Testing

Web Services Testing is a type of software testing that validates the mechanism of communication through which two applications / devices exchange data. The purpose of Web Services Testing is to check the functionality, reliability, performance, and security of an API(Application Program Interface). Testing the services of the application will occur both online and offline and should be irrespective of underlying architecture.

Performance Testing

Performance testing is a testing practice performed to determine how a system performs in terms of responsiveness and stability under a particular workload. In the context of a web application, we will be testing and analyzing the metrics below:

  • User Satisfaction (Apdex)
  • Average response time
  • Error rates
  • Request rate
  • Application and server utilization
  • Application availability
  • Query response time

Security Testing

Due to the sensitive data such as personal information and credit card transactions, a thorough security test will be conducted, Security testing is the process intended to reveal flaws in the security mechanisms of an information system that protect data and maintain functionality as intended. The purpose of adding security testing into the validation process uncovers vulnerabilities of the system and determines that the data and resources of the system are protected from possible intruders. It ensures that the software system and application are free from any threats or risks that can cause a loss.
